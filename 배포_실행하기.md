# READS 배포 — 직접 실행할 명령어

아래 순서대로 **터미널에서 한 번에 복사해 실행**하면 됩니다. (GitHub 사용자명만 본인 것으로 바꾸세요.)

---

## 1단계: GitHub 저장소 만들기

1. 브라우저에서 https://github.com/new 접속
2. **Repository name**: `reads` (또는 원하는 이름)
3. **Public** 선택
4. **Create repository** 클릭 (README, .gitignore 추가하지 않음)
5. 생성된 저장소 URL 확인 (예: `https://github.com/YOUR_USERNAME/reads`)

---

## 2단계: 터미널에서 푸시

아래 블록 전체를 복사한 뒤 터미널에 붙여 넣고, **`YOUR_USERNAME`을 본인 GitHub 사용자명으로 바꾼 다음** 실행하세요.

```bash
cd "/Users/seongyeupkim/Library/Mobile Documents/com~apple~CloudDocs/Docs/채윤/McCarthy project/reads"

git remote add origin https://github.com/YOUR_USERNAME/reads.git
git branch -M main
git push -u origin main
```

- 이미 `origin`이 있으면: `git remote remove origin` 후 위에서 `git remote add` 다시 실행.
- 푸시 시 GitHub 로그인(또는 토큰) 창이 뜨면 본인 계정으로 인증.

---

## 3단계: Vercel 로그인 (한 번만)

```bash
vercel login
```

- 브라우저가 열리면 Vercel 계정으로 로그인.

---

## 4단계: Vercel 배포

```bash
cd "/Users/seongyeupkim/Library/Mobile Documents/com~apple~CloudDocs/Docs/채윤/McCarthy project/reads"
vercel --prod
```

- 처음이면 프로젝트 이름·팀 등 물어보면 엔터로 기본값 선택.
- 끝나면 **Production** URL이 나옵니다 (예: `https://reads-xxx.vercel.app`).

---

## 요약

| 단계 | 할 일 |
|------|--------|
| 1 | GitHub에서 `reads` 저장소 생성 |
| 2 | `YOUR_USERNAME` 바꾼 뒤 `git remote add` + `git push` 실행 |
| 3 | `vercel login` 한 번 실행 |
| 4 | `vercel --prod` 로 배포 후 나온 URL로 접속 확인 |

이후에는 코드 수정 후 `git push`만 해도 Vercel이 GitHub와 연동돼 있으면 자동으로 재배포됩니다 (연동은 Vercel 대시보드에서 설정).
